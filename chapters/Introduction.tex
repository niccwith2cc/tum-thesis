Unfortunately, as revolutionary as autonomous vehicles are, they are not without their flaws. These flaws are indicative of the two main architectural structures which are utilized to develop

The modular architecture also called stack, relies on imposing human written rules to dictate the vehicle's behavior \cite{hu_review_2026, jain_Autonomy_20_2021, jahncke_MINDStackModularInterpretable_2025}.
% perhaps think of a better formulation than: generally made up of so called units which consists
The stack is generally made up of so called units which consists of: perception, takes sensor data to be interpreted and fed into the prediction unit, estimating a future state based on an end goal e.g predicting the path of surrounding cars in traffic.
This prediction allows the planning unit to compute a feasible trajectory which the ego vehicle can perform safely, and ultimately the control unit takes the trajectory and transfers commands to the car in the form of throttle input
and steering angle to traverse the planned path. 

On the one hand, the downstream pipeline offers a very interpretable path for the engineer to understand what to modify in the cost function and ensure well defined
safety measures are implemented \cite{jain_Autonomy_20_2021, jahncke_MINDStackModularInterpretable_2025, karkus_DiffStackDifferentiableModular_2022}. 
However, these rule-based systems depend heavily on experts who continually formulate new rules for the vehicle to find the optimal cost function \cite{jain_Autonomy_20_2021,karkus_DiffStackDifferentiableModular_2022}. 
% edit causing encountering edge cases. I wrote this that way to prevent using "which" too often
Inherently unable to generalize their behavior, causing encountering edge cases or expanding into new locations necessitating extensive re-tuning, which introduces substantial scalability bottlenecks \cite{jain_Autonomy_20_2021,huang_DifferentiableIntegratedMotion_2024, karkus_DiffStackDifferentiableModular_2022}. 
% Edit this line a bit more
Furthermore, implementing a simulator for testing the full stack does not provide an accurate enough evaluation of real world conditions, hence on-road testing will have to be performed, further increasing the scalability bottleneck \cite{jain_Autonomy_20_2021}.

Talk about end-to-end neural network

Alternatively, the end-to-end neural network architecture replaces the entire modular stack, taking in sensor data from the vehicle and through a type of reinforcement learning algorithm attempts to predict the most optimal path and guidance for the ego vehicle.
This requires significant training data to improve the predictions of the neural network. Good at generalizing behavior but struggles with discrete tasks such as safety and road designation.

Talk about MIND stack and how it is important for the racing aspect of autonomous
A middle of the road approach is now picking up traction which combines a modular stack with the end-to-end neural network, called the Modular, Interpretable, End-to-End Differentiable (MIND) stack.
This approach allows to combine the best performance of the modular and neural network stacks with little to no drawbacks
